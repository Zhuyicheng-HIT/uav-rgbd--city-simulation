# 预测避障算法说明

## 输入

- RGB 图像：用于 YOLO 目标检测。
- 深度图：用于估计目标中心点三维位置。
- CameraInfo：用于读取相机内参。
- 无人机位置与姿态：用于把目标从相机坐标转换到局部真实坐标。

默认 D435i 内参可作为 CameraInfo 缺失时的 fallback：

```text
Depth 640x480 K =
[[385.78912354,   0.        , 325.45468140],
 [  0.        , 385.78912354, 237.49458313],
 [  0.        ,   0.        ,   1.        ]]

RGB 640x480 K =
[[606.35186768,   0.        , 328.86410522],
 [  0.        , 605.97052002, 247.47494507],
 [  0.        ,   0.        ,   1.        ]]

D = [0, 0, 0, 0, 0]
```

实际运行时优先使用 `/camera/camera/color/camera_info` 和深度相机对应 CameraInfo，不建议长期依赖写死内参。

## 坐标系

算法使用局部真实坐标系作为目标轨迹和安全点的核心坐标源。

- 视觉激活时，以无人机当前位置的地面投影作为局部地图原点。
- 地面目标检测后从像素坐标 + 深度反投影到相机坐标。
- 根据相机与机身刚性连接关系、无人机姿态和无人机位置，转换到局部真实坐标。
- 目标轨迹、安全点代价函数、上一安全点距离约束都在真实坐标中计算。
- 可视化俯视图由真实坐标反算到屏幕坐标，不把安全点固定在相机画面上。

这样可以避免无人机移动时安全点跟着相机坐标漂移，导致飞机一直向同一方向飞。

## 跟踪

YOLO 每帧输出检测框后，tracker 根据目标中心、深度和历史轨迹进行匹配。

仿真中识别率较高，可以适当收紧同一目标匹配阈值，使同一个目标保持稳定标签。真实场景中建议放宽匹配阈值，并允许目标短暂丢失后继续预测若干帧。

## 安全点生成

算法在局部地图中搜索候选降落点，代价函数综合考虑：

- 与预测目标轨迹的距离。
- 点安全距离。
- 通道安全距离。
- 与上一安全点的距离，减少安全点跳变。
- 与无人机当前投影位置的距离，避免不必要的大范围平移。

距离计算可使用曼哈顿距离降低开销：

```text
distance = abs(dx) + abs(dy)
```

## 输出

- `/vision/avoidance_waypoint`：安全航点。
- `/vision/avoidance_status`：视觉状态、有效性、原因。
- 日志：安全点、目标轨迹、无人机坐标、姿态、帧率等。
